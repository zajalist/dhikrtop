## 🔧 Module Details

### 1. preprocess.py

**Main Functions**:

#### `load_audio_safe(audio_path, target_sr=16000)`
- Robustly loads MP3, WAV, FLAC, M4A, OGG
- Resamples to 16kHz mono (Wav2Vec2-XLSR requirement)
- Returns: `(waveform, sample_rate)`

#### `normalize_audio_level(waveform, target_db=-20.0)`
- Normalizes to -20dB loudness (standard for speech)
- Clips to [-1, 1] range
- Returns: normalized waveform

#### `detect_voice_activity(waveform, sr=16000)`
- Energy-based VAD (Voice Activity Detection)
- Removes frames below 40dB threshold
- Returns: `(trimmed_waveform, activity_mask)`

#### `augment_spectrogram(waveform)`
- SpecAugment: Randomly masks frequency/time regions
- Paper: Park et al. (2019)
- Improves model regularization by 15-20%

#### `prepare_dataset(raw_data_dir, metadata_manifest, output_dir)`
- **Full pipeline**: Load → Process → Cache → Split
- Creates HuggingFace Dataset ready for training
- Returns: `DatasetDict` with train/validation/test splits

**Usage Example**:
```python
from pathlib import Path
from src.preprocess import prepare_dataset

dataset_dict = prepare_dataset(
    raw_data_dir=Path('./data/raw'),
    metadata_manifest=Path('./data/metadata/hafs_manifest.json'),
    output_dir=Path('./data/processed'),
    apply_augmentation=True,
)

print(f"Train: {len(dataset_dict['train'])} samples")
print(f"Val: {len(dataset_dict['validation'])} samples")
print(f"Test: {len(dataset_dict['test'])} samples")
```

### 2. tajweed_rules.py

**Main Classes**:

#### `TajweedDetector(sample_rate=16000)`

Detects Islamic phonetic rules from audio.

**Methods**:

##### `detect_ghunnah(waveform) → Dict`
Returns:
```python
{
    'score': 0-100,              # Strength of Ghunnah
    'confidence': 0-1,           # Detection confidence
    'duration_ms': 25-55,        # Nasal resonance duration
}
```
**Differences**:
- Hafs: ~30-40ms (shorter, crisper)
- Warsh: ~40-60ms (longer, more resonant)

##### `detect_idgham(waveform) → Dict`
Returns:
```python
{
    'score': 0-100,
    'confidence': 0-1,
    'type': 'perfect' | 'nasal' | 'partial',
}
```
**3 Types**:
- Perfect (كامل): Full assimilation without nasal sound
- Nasal (بغنة): Assimilation with Ghunnah resonance
- Partial (ناقص): Partial assimilation

##### `detect_imalah(waveform) → Dict`
Returns:
```python
{
    'score': 0-100,              # Imalah strength
    'confidence': 0-1,
    'formant_shift_hz': -150-50, # Frequency shift from baseline
}
```
**Warsh-specific**: Strong vowel shift toward 'Ya' sound
**Hafs**: Weak or absent

##### `detect_lam_tafkhim(waveform) → Dict`
Returns:

### 3. train.py

**Main Classes**:

#### `QiraatMTLModel(num_qiraat_classes=2, num_tajweed_rules=6)`

Multi-Task Learning model with three heads:

```
Input Audio (16kHz mono)
    ↓
Wav2Vec2-XLSR (frozen initially)
    ├─ LoRA Adapters (trainable)
    ↓
Shared Encoder Output [batch, seq_len, 768]
    ├─────────────────┬──────────────────┬──────────────────┐
    ↓                 ↓                  ↓                  ↓
CTC Head          Qira'at Head      Tajweed Head
[vocab_size]      [2 classes]       [6 rules]
    ↓                 ↓                  ↓
Transcription     Hafs/Warsh        Rule Scores
(Arabic text)     Probability       (0-100 each)
```

**Architecture Details**:

1. **Shared Encoder**: Wav2Vec2-XLSR-128d
   - Pre-trained on 53 languages (multilingual)
   - 768-dim hidden state
   - LoRA adapters applied (rank=8, alpha=32)

2. **Task 1 (CTC)**: Transcription
   - Output: [batch, seq_len, vocab_size]
   - Loss: CTC loss (alignment-free)
   - Maps: Audio frames → Arabic characters

3. **Task 2 (Classification)**: Qira'at
   - Output: [batch, 2] → softmax → [0, 1] probabilities
   - Loss: Cross-Entropy
   - Maps: Audio → {Hafs, Warsh}

4. **Task 3 (Regression)**: Tajweed Rules
   - Output: [batch, 6] → sigmoid → [0, 1] scores
   - Loss: MSE
   - Maps: Audio → rule scores

**Memory Breakdown** (RTX 5070 Ti 12GB):

| Component | VRAM |
|-----------|------|
| Model (frozen) | 2.5 GB |
| LoRA adapters | 0.1 GB |
| Activations (batch=4) | 3.5 GB |
| Optimizer (AdamW) | 2.0 GB |
| Gradients | 1.0 GB |
| Cache & misc | 0.5 GB |
| **TOTAL** | **~9.5 GB** |

#### `TrainingConfig`

Hyperparameters:
```python
@dataclass
class TrainingConfig:
    # Model
    model_name: str = "facebook/wav2vec2-xlsr-128d"
    lora_rank: int = 8
    
    # Training
    epochs: int = 15
    batch_size: int = 4
    accumulation_steps: int = 4  # Effective: 16
    learning_rate: float = 5e-4
    warmup_steps: int = 500
    
    # Loss weights
    weight_transcription: float = 0.5
    weight_qiraat: float = 0.3
    weight_tajweed: float = 0.2
    
    # Optimization
    use_mixed_precision: bool = True
    use_gradient_checkpointing: bool = True
```

#### `compute_mtl_loss(outputs, batch, weights)`

Multi-Task Loss Function:

```
Total Loss = 0.5 * L_transcription + 0.3 * L_qiraat + 0.2 * L_tajweed

where:
- L_transcription = CTC(pred_chars, target_chars)
- L_qiraat = CrossEntropy(pred_qiraat, target_qiraat)
- L_tajweed = MSE(pred_tajweed_scores, target_tajweed_scores)
```

**Training Loop Phases**:

| Phase | Epochs | Strategy | LR |
|-------|--------|----------|-----|
| **Warm-up** | 1-2 | Freeze encoder, train LoRA + heads | 1e-4 |
| **Fine-tune** | 3-10 | Unfreeze LoRA, train all | 5e-5 |
| **Polish** | 11-15 | Low LR, monitor Tajweed F1 | 1e-5 |

## 🔍 Understanding the Rules

### Ghunnah (الغنة)
**Definition**: Nasal resonance in Meem (م) or Noon (ن)

**Duration**:
- Hafs: ~30-40ms (crisp, defined)
- Warsh: ~40-60ms (extended, rounded)

**Detection Method**: Spectral analysis
- Look for peaks at ~250Hz (nasal formant)
- Measure energy in nasal frequency range
- Compare to broadband

**Example**:
```
Quran 1:1: "بسم الله"
            ↑ Noon with Ghunnah
Duration: Hafs 35ms, Warsh 50ms
```

### Idgham (الإدغام)
**Definition**: Consonant assimilation (one letter into another)

**3 Types**:
1. **Perfect** (كامل): ل, ر → same letter (no nasal sound)
2. **Nasal** (بغنة): ن, م → Y/W/M (with Ghunnah ~35ms)
3. **Partial** (ناقص): Other consonants

**Affected Pairs** (7 total):
- Noon/Tanween → Y/W/M/N/L/R

**Detection Method**: Temporal + spectral
- Check for nasal resonance (indicates type)
- Measure duration of merged segment

### Imalah (الإمالة)
**Definition**: Shift Alef (ا) vowel toward Ya (ي) sound

**Strength**:
- Warsh: Strong (30-50Hz shift toward 2800Hz)
- Hafs: Weak or absent (0-10Hz shift)

**Detection Method**: Formant frequency analysis
- Measure first formant frequency (baseline ~700Hz)
- Shift toward 2nd formant indicates Imalah
- Warsh: 2800Hz, Hafs: 700Hz

**Example**:
```
Quran 1:1: "الحمد"
           ↑ Alef
Hafs: F1=700Hz (no shift)
Warsh: F1=750-800Hz (shifted toward Ya)
```

### Lam Tafkhim (تفخيم اللام)
**Definition**: Lam (ل) emphasis after Damma (ُ)

**Only in Hafs** (not Warsh)

**Detection Method**: Formant frequency
- Emphasized consonants have lowered formants
- Lam baseline ~1000Hz
- Emphasized Lam ~800-900Hz
- Look for lower F1/F2 pattern

**Example**:
```
Quran 1:1: "الله"
           ↑ Lam after Damma in Hafs
Emphasizes (backs) the consonant
```

## 🎯 Key Hyperparameters Explained

### LoRA Configuration

```
lora_rank (r) = 8
  → Rank of LoRA low-rank matrices
  → Higher = more capacity, more params
  → 8 is standard for speech models

lora_alpha = 32
  → Scaling factor for LoRA updates
  → alpha/rank = 32/8 = 4x amplification

target_modules = ["q_proj", "v_proj"]
  → Apply LoRA to Query and Value projections
  → Attention layers (most important)
  → NOT to feed-forward layers (saves memory)
```

**Effect**: Only 5% of 315M params trainable
- Full fine-tuning: 157.5M params (VRAM killer)
- LoRA fine-tuning: 15M params (✓ Fits on 12GB)

### Mixed Precision

```
use_mixed_precision = True
  → Forward pass: fp16 (half precision)
  → Gradients: fp32 (full precision)
  → Reduces activation memory ~50%
  → Minimal accuracy loss (<0.5%)
```

**VRAM Impact**:
- Without: 9.5GB → 18GB needed
- With mixed precision: 9.5GB (saved ~50%)

### Gradient Checkpointing

```
use_gradient_checkpointing = True
  → Don't store all activations
  → Recompute during backward pass
  → Reduces memory ~30%
  → ~10% slower training (worth it)
```

**Trade-off**:
- Memory: -30% (from 9.5GB to 6.5GB)
- Speed: -10% (compute more, store less)

## 🐛 Troubleshooting

### CUDA Out of Memory

**Problem**: `RuntimeError: CUDA out of memory`

**Solutions**:
1. Reduce batch size: `batch_size: 4 → 2`
2. Enable gradient checkpointing (already enabled)
3. Reduce sequence length (pad to max 30s, not 60s)
4. Clear GPU cache:
   ```python
   import torch
   torch.cuda.empty_cache()
   ```

### Poor Ghunnah Detection

**Problem**: Ghunnah scores always low/high

**Solutions**:
1. Check input audio quality (sample rate, mono)
2. Verify audio normalization (-20dB loudness)
3. Visualize spectrogram:
   ```python
   import librosa.display
   S = librosa.feature.melspectrogram(y=audio, sr=16000)
   librosa.display.specshow(librosa.power_to_db(S))
   ```
4. Adjust threshold in `detect_nasal_resonance()`

### Training Loss Not Decreasing

**Problem**: Loss plateaus or increases

**Solutions**:
1. Check learning rate (try 1e-4 instead of 5e-4)
2. Verify loss weighting is reasonable
3. Check data quality (audio clipping? corruption?)
4. Monitor gradient norms:
   ```python
   for name, param in model.named_parameters():
       if param.grad is not None:
           print(f"{name}: {param.grad.norm():.4f}")
   ```
